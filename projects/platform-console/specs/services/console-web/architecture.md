## Styling & Theming

Vercel-style design-token system (TASK-PC-FE-038). This is the **canonical**
theming for the operator suite; `web-store` mirrors the same "near-white primary
in dark" approach with its own plain-CSS token set.

- **Design tokens** вҖ” semantic colours are CSS custom properties on `:root` in `app/globals.css` (Geist neutrals as HSL channel triples): `--background` / `--foreground` / `--border` / `--muted(-foreground)` / `--primary(-foreground)` / `--accent(-foreground)` / `--destructive(-foreground)` / `--ring`. `tailwind.config.ts` maps its colour scale to `hsl(var(--token))`, so components use semantic classes (`bg-background` / `text-foreground` / `border-border` вҖҰ) and the whole palette rethemes from one token set.
- **Light / dark** вҖ” `tailwind.config.ts` `darkMode: 'class'`; `globals.css` `.dark` flips the token palette. **next-themes** (`shared/ui/ThemeProvider`, wrapped in the root `app/layout.tsx`) drives the `.dark` class on `<html>` with `attribute="class"`, `defaultTheme="system"`, `enableSystem`, `disableTransitionOnChange`, and injects a pre-hydration script so the correct theme paints with no flash (root `<html suppressHydrationWarning>`). `shared/ui/ThemeToggle` (top bar, `(console)/layout.tsx`) flips light/dark and renders a mount-guarded sun/moon icon to avoid an SSR/client hydration mismatch.
- **Shared table style** вҖ” the `.data-table` component class (`globals.css`, TASK-PC-FE-042) gives every console table the rounded, bordered Vercel look from one class: `border-separate` + `border-spacing-0` + `overflow-hidden` clips the rounded outer frame (the collapsed-border model ignores `border-radius`), and horizontal dividers move to cell `border-b` since the separate-borders model ignores per-`<tr>` borders.

### Active-tenant мҠӨмң„мІҳ вҶ’ assume-tenant көҗнҷҳ (ADR-MONO-020 D4 / `console-integration-contract.md` В§ 2.7)

мҪҳмҶ”мқҖ **л‘җ к°ңмқҳ server-side RFC 8693 көҗнҷҳ**мқ„ к°Җм§„лӢӨ (нҳјлҸҷ кёҲм§Җ):

1. **operator-identity көҗнҷҳ** (ADR-MONO-014, В§ 2.6): admin **JSON** `POST /api/admin/auth/token-exchange` вҶ’ `/api/admin/**` operator token. ліҖкІҪ м—ҶмқҢ.
2. **assume-tenant көҗнҷҳ** (ADR-MONO-020 D4, В§ 2.7, **мӢ к·ң**): SAS **form-urlencoded** `POST ${OIDC_ISSUER_URL}/oauth2/token`, `grant_type=...token-exchange` + `subject_token=<base IAM OIDC access token>` + `audience=<м„ нғқ tenant>`. вҶ’ м„ нғқлҗң customer лЎң re-scope лҗң лӢЁлӘ… domain-facing IAM OIDC token (`tenant_id=<м„ нғқ>` + `entitled_domains=<м„ нғқмқҳ ACTIVE subs>`). **refresh_token м—ҶмқҢ** вҖ” selection / IAM refresh л§ҲлӢӨ мһ¬л°ңкёү. producer кі„м•ҪмқҖ IAM [`auth-api.md В§ Assume-Tenant Exchange`](../../../../iam-platform/specs/contracts/http/auth-api.md) (TASK-BE-327) мҶҢмң  вҖ” мҶҢл№„л§Ң, мҷҖмқҙм–ҙ shape мһ¬м •мқҳ кёҲм§Җ.

**assumed-token м„ём…ҳ + domain-facing resolver**: `shared/lib/session.ts` м—җ `ASSUMED_TOKEN_COOKIE`(`console_assumed_token`, HttpOnlyВ·SecureВ·Lax) + `getAssumedToken()` + **`getDomainFacingToken()` = `getAssumedToken() ?? getAccessToken()`** (мӨ‘мӢ¬ resolver). лӘЁл“  tenant-scoped лҸ„л©”мқё read мқҳ IAM-OIDC bearer = `getDomainFacingToken()`:

- cross-domain overview н”„лЎқмӢң(В§ 2.4.9.1) `Authorization: Bearer` (BFF мқҳ non-IAM fan-out leg мқҙ verbatim forward; IAM leg мқҖ `X-Operator-Token` мң м§Җ вҖ” В§ 2.6 л¶ҲліҖ);
- 4 non-IAM лҸ„л©”мқё нҒҙлқјмқҙм–ёнҠё(`features/{wms,scm,finance,erp}-ops`, В§В§ 2.4.5вҖ“2.4.8) вҖ” per-domain credential к·ңм№ҷ В§ 2.4.5 л¶ҲліҖ, **м–ҙлҠҗ IAM OIDC token** мқём§Җл§Ң ліҖкІҪ.
- **IAM лҸ„л©”мқё нҒҙлқјмқҙм–ёнҠё**(`features/{accounts,audit,operators,dashboards}` вҶ’ `getOperatorToken()`)лҠ” **ліҖкІҪ м—ҶмқҢ** вҖ” operator-token кІҪкі„(В§ 2.1/В§ 2.6, #569)лҠ” к·ёлҢҖлЎң. `getDomainFacingToken()` мқҖ кІ°мҪ” IAM `/api/admin/**` мһҗкІ©мқҙ м•„лӢҲлӢӨ.
- **net-zero**: лҜём „нҷҳ / single-tenant мҡҙмҳҒмһҗлҠ” assumed token мқҙ м—Ҷм–ҙ `getDomainFacingToken()` = base token вҶ’ кё°мЎҙ лҸҷмһ‘ byte-identical.

**мҠӨмң„м№ҳ лқјмҡ°нҠё(`POST /api/tenant`)**: кё°мЎҙ registry-membership allow-check (defence-in-depth вҖ” мң м§Җ) нӣ„ assume-tenant көҗнҷҳ нҳём¶ң; м„ұкіө мӢң `console_assumed_token`(maxAge=`expires_in`) + `console_active_tenant=<tenant>` лҘј **atomically** м„Өм •. assumed token мқҖ нҳ„мһ¬ active tenant лЎң construction-scoped. **fail-closed**: `denied`(producer `400 invalid_grant` вҖ” D2 assignment кІҢмқҙнҠё/subject л¬ҙнҡЁ/admin-service мһҘм• ) вҶ’ 403, **мҝ нӮӨ л¬ҙліҖкІҪ**(мқҙм „ м„ нғқ ліҙмЎҙ); `invalid`(`400 invalid_request`) вҶ’ 422; `unavailable`(5xx/timeout/network) вҶ’ 503; base token л¶Җмһ¬ вҶ’ 401. base IAM token мқҖ `subject_token` мһ…л Ҙмқј лҝҗ вҖ” **лҜёлЎңк№…В·лҜёл°ҳнҷҳ**; м„ нғқ-tenant кІҪкі„м—җм„ң base token мңјлЎң **м ҲлҢҖ fallback м•Ҳ н•Ё**. `tenant=''` clear вҶ’ **л‘җ мҝ нӮӨ лӘЁл‘җ** мӮӯм ң(coupled).

**refresh мһ¬-assume**: assumed token мқҖ refresh token мқҙ м—ҶмңјлҜҖлЎң IAM refresh мӢң operator-token мһ¬көҗнҷҳ нӣ„ active tenant к°Җ мһҲмңјл©ҙ rotated base token мңјлЎң **мһ¬-assume**. мһ¬-assume мӢӨнҢЁ вҶ’ assumed token + active tenant лӘЁл‘җ drop (base/no-tenant лЎң fallback вҖ” stale assumed token кёҲм§Җ); base IAM+operator м„ём…ҳмқҖ мң нҡЁ. м „мІҙ м„ём…ҳ drop(IAM refresh кұ°л¶Җ / operator мһ¬көҗнҷҳ fail-closed) мӢңм—җлҸ„ assumed token + active tenant clear.

**м„ нғқ к°ҖлҠҘ tenant**: мҠӨмң„мІҳмқҳ м„ нғқ к°ҖлҠҘ м§‘н•© = ConsoleRegistry effective scope (TASK-BE-326 dual-read: assignment rows вҲӘ legacy home tenant) вҖ” multi-assignment мҡҙмҳҒмһҗлҠ” н• лӢ№лҗң лӘЁл“  customer к°Җ л…ём¶ң (BE-326 к°Җ мқҙлҜё wire вҖ” мҪҳмҶ” ліҖкІҪ л¶Ҳн•„мҡ”). D4 лҠ” selection мӢң credential re-scope л§Ң.
